Appcelerator Cloud Services (ACS for short) provides a wide array of automatically-scaled network features and data objects for your application. Tasks such as push notifications, user logins, photo uploads, checkins and status updates that usually require server programming or tedious integration with multiple SDKs are performed through one simple interface. You pick and choose which APIs to use and we take care of deploying and maintaining a full server stack that includes a database, search engine, file storage, and application logic.

For example, you can:

  • Create a server backend for your app instantly

  • Launch and scale your app automatically

  • No server coding or administration required

For more information, see Integrating with Appcelerator Cloud Services.

Adding ACS to a new project

In project creation, a checkbox appears (checked by default) on the Titanium Mobile project creation screen with the text "Automatically cloud-enable this application"

If you are offline, the checkbox is disabled, and you can enable their application later once they are online.

The checkbox is only enabled with SDKs 2.0+. If they have an SDK < 2.0, you need to chose a later SDK.

At the end of the project creation wizard, a new cloud app is registered with two API keys

Adding ACS to an existing project

On the tiapp.xml editor screen, note the button to "Enable Cloud Services". Once enabled, the same "development" and "production" keys are used as before.

Using ACS in your application

At this point, use the Ti.Cloud platform SDKs to add ACS calls into your application. See the platform SDK documentation for more information.

Switching between development and production keys

No action is necessary on the part of the developer. The "development" key is used automatically when running or debugging the application in an emulator or simulator. The "production" key is used when deploying to device

Removing ACS from a project

  1. Switch to the source view of tiapp.xml

  2. Delete the properties referencing the ACS keys - properties named acs-*

  3. Save the editor

You may re-add ACS at any time by following the same steps as adding ACS to an existing project.

Troubleshooting

Error enabling Cloud service for project

If you receive a message similar to "Error enabling Cloud service for project" in Studio when trying to enable Cloud services for your project, the ACS server may be down or Studio is unable to connect to the ACS server. Try to enable Cloud services later.

If you are still receiving this error when trying to enable Cloud services, check to see if your application was created in either https://my.appcelerator.com/apps (for an application created in Titanium Studio) or https://dashboard.appcelerator.com/ (for an application created in Appcelerator Studio). If the application was created, check to see if ACS keys were created for the application. To find your ACS keys:

  • If you are using Titanium Studio (free, community version), go to https://my.appcelerator.com/apps and click on the Manage ACS link below your application. You should see an item for APP KEY. Note that you need both the Development and Production version. You can switch between the two by pressing the Development and Production buttons in the top-right corner of the web page.

  • If you are using Appcelerator Studio (paid, enterprise version), go to https://dashboard.appcelerator.com/, select your application from the Apps drop-down list, click the Cloud tab, then click Settings & Configuration. You should see an item for App Key. Click the Show link to expand these items. Note that you need both the Development and Production version. You can switch between the two by clicking the drop-down box in the top-right corner that displays either Development or Production.

If you do not have ACS keys, try to enable Cloud services again at a later time.

If you do have ACS keys, manually enter the ACS key information in to the tiapp.xml file. To manually enter this information:

  1. Double-click your tiapp.xml file to open it in the Editor.

  2. Click the tiapp.xml tab in the lower-left corner of the Editor.

  3. Insert the following ACS property keys as children of the ti:app parent tag and replace with the application's ACS keys found earlier:

    <?xml version="1.0" encoding="UTF-8"?>
    <ti:app xmlns:ti="http://ti.appcelerator.org">
    <!-- Add these elements and replace with your application's ACS keys -->
    <property name="acs-api-key-development" type="string">APP_KEY_DEV</property>
    <property name="acs-api-key-production" type="string">APP_KEY_PROD</property>
    <!-- Add this element if you are using Appcelerator Studio -->
    <property name="acs-base-url" type="string">https://api.cloud.appcelerator.com</property>
    ...
    </ti:app>

  4. Save and close your tiapp.xml file.

  5. Reopen your tiapp.xml file. In the Overview tab, it should show that Cloud services is enabled.
