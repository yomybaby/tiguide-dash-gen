Overview

The following is a walk-through that covers setup and retrieval of analytics data.

Terminology

Analytics

Analytics refers to data about the way an application is used. Analytics is collected by Appcelerator for the Titanium developer of the app. Some analytics data can be viewed using the Analytics UI. Other analytics data is only available to users that sign up for the optional raw data export service. Raw data export provides for a daily download of raw analytics data.

Events

These are operational milestones in the application. Some events are standard, such as those that mark the beginning and end of a session. Others may be custom events which have a meaning specific to a single application.

Feature Event

A feature event is a custom event that has been defined by the developer of a given application. A feature event should represent an action, such as 'new item', 'launch video', etc. All feature events should follow a naming convention group.event.sub-event with group.event being the parent event, under which sub-events are grouped in the Analytics UI. For example, a parent event tweet.new may have sub-events tweet.new.cancel and tweet.new.post .

Use the Titanium.Analytics.featureEvent method to generate a feature event.

An arbitrary data object can be added to each feature event, however this data is not accessible through the Analytics UI, only through raw data export.

Navigation Event

A navigation event is a custom event that is used by the application to track movement from one area of the UI to another.

Navigation events are not currently available through the Analytics UI, only through raw data export.

Use the Titanium.Analytics.navEvent method to generate a navigation event.

Setup

  • Create a new Mobile Application using the account that you want to use to see your analytics data.

    • An app is tied to the account that created it. Analytics for the app is accessible from that account's my.appcelerator.com/apps page.

    • To verify that an app was create by the correct account

      • Open up tiapp.xml and look for the app "guid"

      • Log into my.appcelerator.com/apps, search your apps for the "App GUID" that you found in the tiapp.xml. You can see the App GUID by clicking "View Details" for your app.

  • By default analytics will be enabled. To verify, make sure that tiapp.xml has this line <analytics>true</analytics>

  • Track a custom action in your application using Titanium.Analytics.featureEvent(NAME,[DATA]).

Code Sample

Here is an example of modifying the Tweetanium app to send feature events.
in DrawerView.js:

//add logic to manage the state of the drawer
Ti.App.addEventListener('app:show.drawer', function(e) {
drawerView.visible = true;
//determine which view to show in the drawer
if (e.showing === 'createAccount') {
stack.fireEvent('changeIndex',{idx:0});
}
else if (e.showing === 'tweetDetails') {
Titanium.Analytics.featureEvent('tweet.details');
stack.fireEvent('changeIndex',{idx:1});
}
else if (e.showing === 'accountDetails') {
stack.fireEvent('changeIndex',{idx:2});
}
drawer.animate({
duration:$$.animationDuration,
left:10
}, function() {
Ti.App.fireEvent('app:drawer.opened');
});
});

in ComposeWindow.js:

cancelButton.addEventListener('click', function() {
Titanium.Analytics.featureEvent('tweet.new.cancel');
win.close();
});
tweetButton.addEventListener('click', function() {
Titanium.Analytics.featureEvent('tweet.new.post',{message:tweetField.value});
Ti.App.fireEvent('app:show.loader');
tt.app.currentAccount.updateStatus({
status:tweetField.value,
success: function(jsonResponse) {
// Ti.API.info(jsonResponse);
alert('Tweet posted!');
Ti.App.fireEvent('app:hide.loader');
win.close();
}
});
});

As explained above, tweet.new.post and tweet.new.cancel will be grouped together under tweet.new, while statistics for tweet.details will be in a separate group (of the same name).

In the example, the tweet.new.post feature event has an object as its data payload. This payload does not have any bearing on the Analytics UI, but in the near future we will provide a mechanism to retrieve this data for post-processing by the developer.

Best Use - Feature Events

Bad
Titanium.Analytics.featureEvent('Select Item THIS IS THE DESCRIPTION OF THE EVENT -12345678910');
Good
Titanium.Analytics.featureEvent('select.item',
{
id: '12345678910',
desc: 'THIS IS THE DESCRIPTION OF THE EVENT'
}
);

Alternatively:

Less Bad
Titanium.Analytics.featureEvent('select.item.12345678910');

Which is appropriate in cases where you want to see a sub-event for each item. Be aware that this currently has very bad performance characteristics in the UI when you create many sub-events. We're working on making this better.

Best Use - Nav Events

Do not use feature events to track movement between navigational elements of your application. Instead, use a nav event to collect such analytics.

Very Bad
Titanium.Analytics.featureEvent('Home -> Edit Tweet');
Good
Titanium.Analytics.navEvent('Home', 'Edit Tweet');

Packaging

  • In order for your app's analytics data to show up you need to package your app for distribution.

    • This means that you will need to click "Distribute - App Store" or "Distribute - Android".

    • The type of distribution "ad-hoc" or "App Store/Market" does not matter.

Running

  • Install your app on a device, launch it, close it, and do something that will feature an event.

Analytics UI

Apps that are in production (that is, they have been packaged and installed on devices 'in the wild') have their analytics data accessible via the Analytics UI.

  • Go to https://my.appcelerator.com/apps and click on the "Analytics" button for the app. A short cut URL uses the application's GUID: 

    http://analytics.appcelerator.net/#app=GUID

  • Analytics data from the previous day becomes available in the Analytics UI sometime in the early morning of U.S. Pacific time. (Analytics data for a given day is processed the next day sometime after midnight U.S. Pacific TIme.)

  • Feature events for your app will be visible in the Events section.

  • In the near future, a section displaying nav events will be added.
