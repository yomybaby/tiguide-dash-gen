Currently, Windows 8 App development with the Titanium SDK is restricted with the following conditions:

  • All development is done through the command-line tools. No support with Titanium or Appcelerator Studio.

  • You can only test applications locally on the same machine you are developing on. No support for testing remotely or with the simulator.

Overview

This guide walks through the steps to download and setup the Windows 8 SDK to support Windows App development using the Titanium Mobile Web API. The Titanium SDK turns your Mobile Web applications in to Windows App JavaScript projects. With these projects, you can run, test and package your application in Microsoft Visual Studio. Windows Apps are Windows Store applications that run on x86, x64 and ARM devices running Windows 8. Titanium does not support Windows Store applications running on RT devices.

For Windows Phone development, see Getting Started with the Windows Phone SDK.

Compatibility and Download

Titanium requires the Window 8 SDK and Visual Studio to develop Windows Apps.

Windows 8 SDK and Visual Studio

Microsoft only supports Windows Store development on systems running Windows 8. Titanium does not support for Windows 8.1.

Windows App development requires Visual Studio, which includes the Windows 8 SDK. Titanium only supports Visual Studio Pro (or higher) and either the 2012 or 2013 edition.

  • Visual Studio 2012 supports the Windows 8 SDK

  • Visual Studio 2013 supports the Windows 8.1 SDK

Windows Dev Center Account

To publish applications, you need a Windows Dev Center Account. See http://msdn.microsoft.com/windows/.

Download a trial version of Visual Studio Professional 2013 from http://www.microsoft.com/visualstudio/downloads.

Installation

Before installing any components, make sure your Windows OS is up-to-date with the latest updates. If you have an existing version of Visual Studio installed, make sure it is up-to-date with the latest updates.

Download and install Visual Studio:

  • Visual Studio needs to be installed to the default location: C:\Program Files (x86)\Microsoft Visual Studio <VERSION>.

  • Visual Studio Professional 2012 installs the Windows 8 SDK during setup.

  • Visual Studio Professional 2013 installs the Windows 8.1 SDK during setup.

Launch Visual Studio at least once before building an application to setup the Visual Studio environment.

Setting PowerShell Permissions

Before executing any commands from the terminal, you need to change the execution policy for PowerShell to run some of the build scripts. This process only needs to be done once.

  1. Open PowerShell as an Administrator:

    1. Go to the Start screen or Apps view.

    2. Start typing "PowerShell".

    3. When the Windows PowerShell icon appears, right-click it and select Run as administrator.

  2. Execute the following command:

    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

The previous command enables the execution of all scripts that are signed by a trusted publisher for the current logged in user.

For more information about this command, see Microsoft TechNet: SetExecution-Policy article.

Using the CLI

To build a Windows App, execute the following command:

titanium build -p mobileweb -T winstore

After the build process is complete, your application launches automatically. A launch icon for your application appears in the Apps view on your computer.

This build process converts the Titanium project to a Windows App project that can be opened in Visual Studio.

First Time Building an Application

The first time you try to build an application. You will be prompted by a dialog to create a private key and certificate. Enter a password and confirm it. After the dialog is dismissed, you will be prompted twice for the same password later in the build process. You will only be prompted for this information once. Subsequent builds should not ask you for the password.

You will also be prompted to create a Windows Developer Certificate by a dialog. Accept the request and a developer certificate will be installed to your computer. Subsequent builds should not ask you to join.

Using Visual Studio

Once you have built you project, you can open it in Visual Studio to take advantage of its debugging tools and package it for distribution on the Windows Store.

Opening the Project in Visual Studio

  1. From the menu, select File > Open > Project/Solution or press Control + Shift + O to open the Open Project dialog.

  2. Navigate to your project's ./build/mobileweb-winstore directory.

  3. Select your project's SLN file and click Open.

Updating the App Manifest File

Double-click the package.appxmanifest file in the Solutions Explorer view (equivalent of the App Explorer view in Eclipse-based IDEs) to open it in the Manifest Designer (equivalent of the Editor in Eclipse-based IDEs).

This file is used to manage settings used for deployment, such as specifying icon assets, orientation modes and application permissions.

Packaging the Application

To package your application, from the menu, select Project > Store > Create App Packages... and follow the dialog directions.

API Usage and Limitations

All of the Mobile Web APIs work for Windows Phone and Store applications with the following exceptions:

  1. You cannot use the HTTP client to make requests back to yourself, that is, you cannot use localhost. Because of this limitation, the RSS Reader sample application does not work.

  2. You cannot use protocol relative URLs, for example, //www.appcelerator.com/support. The default protocol for Windows Phone and Store applications is neither HTTP nor HTTPS.

  3. For Windows Store applications, when using the alert method, if the application currently has an alert dialog open, a new alert cannot be opened until the current one is dismissed.

Windows API

This section describes APIs provided by Microsoft. These APIs are not supported by Appcelerator and are not guaranteed to work with the Titanium SDK.

The Windows 8 SDK also provides a JavaScript API for it applications, which can be utilized in your Windows Store applications. See the following Microsoft guides:

  • Developing Windows Store apps (JavaScript)

  • Windows API reference for Windows Store apps

  • JavaScript Language Reference (Description of Microsoft's JavaScript engine implementation)

Check the Requirements section at the bottom of each API reference for the "Minimum supported client" to see which version of Windows the API supports.

The Windows API provides two sets of APIs with two different main namespaces:

  • The Windows namespace is for JavaScript, C#, C++ and Visual Basic APIs. Not all APIs can be accessed using JavaScript. Check the Syntax section of the API reference to see if it provides a JavaScript example.

  • The WinJS namespace is for JavaScript only.
