Keep it simple and consistent

Try and avoid complex formatting , and keep the tone simple and easy to understand, with a minimum of jargon.

Give it a Table Of Contents and a Summary

Placing a Table Of Contents (toc) and a Summary right at the top of the page makes the guide much more intuitive and easy to navigate. Thus, the following codeblock should be added to every guide.

h6. Chapters
{toc:style=circle|minLevel=2|maxLevel=5}
h2. Summary
This guide explains...
In this guide you will learn:
* how to 1...
* how to 2...
* how to 3...

Structure your document using headings h2 to h5

Giving your documents a logical structure in the form of section headings makes them easier to follow. This is done using the h2 to h5 heading macro codes, which translate to their respective HTML tags. These headings are automatically used to update the Table Of Contents, which provides an essential navigation tool for the reader.

As an HTML h1 tag is already used for the page title, headings within the document should range from h2 to h5. Macro code h6 is reserved for the Chapters heading, so that it does not appear in the Table of Contents.

Hence, the following headings should be used:

h2. : This is heading 2, used for the top-level section headings
h3. : This is heading 3, used for first sub-section headings
h4. : This is heading 4, used for second sub-section headings
h5. : This is heading 5, used for third sub-section headings

Emphasize important information using panels

The info, note, tip and warning panels are great for bringing the user's attention to something outside the main flow of the text.

A tangental point, related to the main text, that the reader may find useful

A related point that should be noted or emphasized

A relevant suggestion that may make life easier for the user

A related issue or gotcha that the user should be aware about

Use the code macro for syntax highlighting

Surrounding your code with the code macro code escapes all of the content, meaning that it will not interfere with the rest of the page. Furthermore, you can set the language used for syntax highlighting, using the argument code:lang=JavaScript, which makes it much easier to read:

var window = Titanium.UI.createWindow();
window.open();

In addition to JavaScript, the XML, HTML and SQL values are also available. If the language you wish to display is not supported, use the code macro without any arguments, or disable syntax highlighting altogether using code:lang=none.

Writing Navigation Instructions

When writing about how to navigate the Studio user interface, bold terms that refer to menu options and buttons:

# In Firefox, select *Extensions* (called *Add-ons* in some Firefox configurations) from the *Tools* menu to open an *Extensions* pop-up window.
# To the right of the path for the *Error Log*, click the *View* button to display the Error Log.

displays as:

  1. In Firefox, select Extensions (called Add-ons in some Firefox configurations) from the Tools menu to open an Extensions pop-up window.

  2. To the right of the path for the Error Log, click the View button to display the Error Log.

Embedding Images

When formatting images, you can add a little extra space to make them separate visually on the page. This is especially useful if you wish to inline an image in a bulleted list

* Item 1
* Item 2 \\ !Screen shot 2011-03-29 at 8.53.25 AM.png|vspace=10!
* Item 3

Which results in:

  • Item 1

  • Item 2

  • Item 3
