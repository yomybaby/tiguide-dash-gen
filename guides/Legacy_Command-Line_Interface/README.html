The titanium.py script in the Mobile SDK provides a command line interface to various common tasks such as creating, building, and running Titanium Mobile projects and modules.

For creating and building projects, you can also use the new Titanium CLI (see Titanium Command-Line Interface Reference). In this release, the legacy CLI is still required for creating and building module projects.

To run apps in the iOS simulator or package apps for iOS distribution, you must use a separate, iOS-specific builder script.

Environment Setup

It's recommended that you setup an alias or add the SDK to your PATH so you can call it directly.

If you're installing the Titanium Mobile SDK from Titanium Studio, the location of the Mobile SDK is controlled by the Titanium SDK Home preference setting. To find this setting in Titanium Studio:

  1. Click Window > Preferences. On OS X, use Titanium Studio > Preferences instead.

  2. Expand the Titanium Studio item and click on Titanium.

  3. Check the value of Titanium SDK Home.

If you did not install the Mobile SDK using Studio, use the path where you installed the Mobile SDK.

The following sections provide steps for setting up the environment on different development OSes:

Windows

Open your environment variable settings by going to:

  • Start Menu > Right click on "Computer" > Properties > Advanced System Settings

  • Click on the "Advanced" tab, and the "Environment Variables" button near the bottom

  • Double click or create a new entry for PATH under User Variables.

  • in Window Vista or Windows 7:

    • Add the Mobile SDK path you looked up in Titanium Studio.

    • If you didn't install the SDK from Studio, use the path where you installed the SDK, such as:
      C:\ProgramData\Titanium\mobilesdk\win32\<VERSION>
      Where <VERSION> is the version of the SDK you have installed.

  • In Windows XP:

    • Add the Mobile SDK path you looked up in Titanium Studio.

    • If you didn't install the SDK from Studio, use the path where you installed the SDK, such as:
      C:\Documents and Settings\All Users\Application Data\Titanium\mobilesdk\win32\<VERSION>
      Where <VERSION> is the version of the SDK you have installed.

OS X

Add aliases for titanium.py and builder.py to your ~/.bash_profile, or create a ~/.bash_profile file if you don't already have one.

Depending on where your mobile SDKs are installed, this will probably look like:

alias titanium.py="/Library/Application\ Support/Titanium/mobilesdk/osx/<VERSION>/titanium.py"
alias ios_builder="/Library/Application\ Support/Titanium/mobilesdk/osx/<VERSION>/iphone/builder.py"

or

alias titanium.py="$HOME/Library/Application\ Support/Titanium/mobilesdk/osx/<VERSION>/titanium.py"
alias ios_builder="$HOME/Library/Application\ Support/Titanium/mobilesdk/osx/<VERSION>/iphone/builder.py"

Where <VERSION> is the version of the SDK you have installed.

Note that previous versions of these instructions suggested using `titanium` as an alias for the `titanium.py` script. This is no longer recommended because it conflicts with the new Titanium CLI. In the current release, the legacy CLI is still useful for creating and building module projects.

The builder.py script is in the iphone subdirectory of the SDK folder.

To add the alias to your environment, run the following command:

source ~/.bash_profile

Linux

Add this to your ~/.bash_profile

alias titanium.py=$HOME/.titanium/mobilesdk/linux/<VERSION>/titanium.py

Where <VERSION> is the version of the SDK you have installed.

Commands

Command

Description

create

Creates a Titanium Mobile project (use --type=module for module projects).

create [--platform=p] [--type=t] [--dir=d] [--name=n] [--id=i] [--ver=v]
--platform=p1,p2 platform: iphone, ipad, android, blackberry, etc.
--type=t type of project: project, module, plugin
--dir=d directory to create New the project
--name=n project name
--id=i project id (ie com.companyName.project)
--ver=i platform version
--android=sdk_folder For android module - the android sdk folder

run

Runs a Titanium Mobile project or module "example" project.
Note: the iPad simulator cannot be run directly from titanium.py.

run [--dir=d] [--platform=p] [--type=t]
--dir=d project directory
--platform=p platform: iphone, android, mobileweb
--type=t type of project: project, module

emulator

Starts the Android emulator.

docgen

Generates HTML documentation from Markdown in an Android module.

docgen [--dir=d] [--dest-dir=d]
--dir=d project directory
--dest-dir=d destination directory

fastdev

Management interface for the Android Fastdev server.

help

Shows help command

iOS Builder Commands

The iOS builder.py script can be used to run apps using the iPad simulator and build apps for distribution.

Run an app on iOS simulator:

ios_builder run <full_path_to_project_folder> <ios_sdk_version> <app_id> <app_name> [ ipad | iphone ]

Where:

  • <app_id> is the Titanium/iOS app ID (for example, com.example.myapp).

  • <app_name> is the application name (for example, MyApp).

Build an app for distribution:

ios_builder distribute <ios_sdk_version> <full_path_to_project_folder> <app_id> <app_name> <provisioning_profile_id> <developer_id> <full_path_to_project_folder>/build/iphone/ [ ipad | iphone | universal ]

Where:

  • <app_id> is the Titanium/iOS app ID (for example, com.example.myapp).

  • <app_name> is the application name (for example, MyApp).

  • <provisioning_profile_id> is the ID of the distribution provisioning profile (find this in the Xcode Organizer window).

  • <developer_id> is the ID of the developer, the string following "iPhone Developer:" on the Developer Profile tab of the Xcode Organizer window.

Examples

Create an Android Project
$ titanium.py create --platform=android --android=/path/to/android-sdk --id=com.mycompany.myApp --name=myApp
Run a project on the Android Emulator
$ titanium.py run --platform=android
