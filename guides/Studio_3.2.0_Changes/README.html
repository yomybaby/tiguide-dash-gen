Introduction

This guide covers the user interface changes to Studio 3.2.0 due to the upgrade from Eclipse 3.7 (Indigo) to 4.3 (Kepler) and assumes you are familiar with the basic components of an Eclipse-based IDE. This guide also covers new features and breaking changes introduced in Studio 3.2.0.

Workbench

The screenshot below compares Titanium Studio 3.1.3 (left side in the background) to Titanium Studio 3.2.0 (right side in the foreground). Both Studios are using the Blackboard editor theme in the Studio (extended) Perspective. The default workbench theme for Studio 3.1.3 is called Default, while for 3.2.0, it is called Dashboard. These two themes are similar and used in the screen shot below. The 3.2.0 theme called Default matches the new Eclipse 4.3 style.

The following list describes some of the significant changes to the default appearance of the workbench:

  • Many of the icons from the global tool bar have been removed to simplify the appearance and streamline the usage for mobile application projects, as well as a new Global Search bar (with the Quick Access hint text). Compare the toolbar across the top of the workbench.

  • The currently selected view or editor is indicated with a brighter white tabbed bar. Compare the tabs in the Editor .

  • The appearance of editor and non-editor views are now independently controlled. Compare the Explorer views.

To change the appearance of the workbench and non-editor views, open the Preferences dialog and select General > Appearances to choose a different theme or use the Colors and Fonts to customize specific items. You can also set the overall theme from Studio > Themes, then selecting an new theme from the Overall Theme drop-down list.

To change the appearance of the Editor, open the Preferences dialog and select Studio > Themes and in the Editor Theme section, choose a different theme or to customize specific items. For more information, see Modifying Editor Colors.

Launch Modes

To launch a project, first select the project in the Project Explorer view, then in the global tool bar select the launch mode and device to launch the application on from the Launch Mode drop-down list and Target drop-down list, respectively. If the Launch Automatically option is enabled under the Target drop-down list, the application will be automatically launched after the target is selected. If not, you need to click the Launch button to start the build process.

To use a launch configuration, first select the launch mode, then select the <Launch> Configuration... option under the Target drop-down list.

If you do not see your device or simulator listed in the Target drop-down list, select Refresh Device from the Target drop-down list.

Previously, these options were separated into different launch mode buttons (Run, Debug, Package, etc.) in the App Explorer and Project Explorer tool bar.

Global Search Bar

The workbench now has a global search field in the global tool bar. Use this search bar to search for Studio commands, views, preferences, etc. This search does not perform a content search on files.

Multi-Adjustable Views

You can now place the cursor at the intersection of two or more views and drag them all at the same time to readjust the layout. When dragging multiple views, the cursor changes to a crosshair-style cursor. The image below indicates where you can readjust the layout with a red circle.


App Explorer

The App Explorer view is deprecated in this release and will be removed in a future release of Studio. Use the Project Explorer view instead.

Project Explorer

As noted earlier, all launch mode buttons have been removed from the Project Explorer toolbar and is now selected through the global tool bar. The only mode you can control in the Project Explorer is LiveView if you have Appcelerator Studio.

Platform Configuration Wizard

Studio 3.2.0 introduces a new platform installation and configuration tool to make it easier to install and update the required platform SDKs to for cross-platform mobile application development. For more information, see Installing Platform SDKs.

Editor

Detached Editors

Editors can now be detached into their own windows. The detached editor is treated the same as an editor contained within the workbench window. Multiple editors and views may be detached and stacked together. The screen shot below displays the three detached editors in the foreground with two editors grouped together.

Saturated Colors in Ruler

You can change the saturation of the ruler in the editor for easier visibility. The below screenshot compares Studio 3.1.3 (left, foreground) to Studio 3.1.4 (right, background). This option is turned off by default and can be enabled in the Preferences dialog, selecting the General > Editors > Text Editors > Accessibility and enabling the Use saturated colors in overview ruler option.


Debug Perspective and View

When using the Debug Perspective. The common debug control commands (Stop, Resume, etc.) have been moved out of the Debug View and into the global toolbar.

The global toolbar can be customized with the Window > Customize Perspective... action to add or remove debugging commands, or to show or hide the debug toolbar completely.

The old toolbar behavior can be restored by first toggling the Show Debug Toolbar option in the Debug view's menu (icon of the white arrow point down), then by disabling the Debug toolbar from the Customize Perspective dialog.

Other Changes

Import Projects Wizard

The Import Projects wizard provides a new Search for nested project option to continue searching for projects to import recursively within any project it finds, allowing you to import physically nested projects at the same time.

Launch Configurations Dialog

In the launch configurations dialog (Run, Debug, Publish, etc.), the dialog warns you if you try to name a custom configuration with the same name as an existing one. Previously, the dialog did not display a warning and not prompt to save the name change.

Open Resources Dialog

The Open Resources dialog provides a new Show In... button to open resources in a Studio View rather than an Editor.

Search Dialog

The Search dialog provides a new Whole word option when searching files and a new Remember last used page option in the Customize... dialog to open the last used page rather than the best matching page.
