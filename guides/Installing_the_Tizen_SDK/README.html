Overview

This guide walkthroughs the steps to download and setup the Tizen SDK and tools as well as how to configure the Titanium environment to support Tizen development.

Compatibility and Download

Titanium requires the Tizen SDK to develop Tizen applications. The Tizen SDK does not officially list Windows 8 as a supported OS, but preliminary testing with the Titanium SDK using Windows 8 and the 32-bit version of the Tizen SDK has not shown any issues.

Tizen development requires Titanium SDK 3.1.0 and later, and Studio 3.1.0 and later. The Tizen SDK requires Oracle JDK v6. The Oracle JDK should already be installed since this is also a prerequisite for the Titanium SDK.

For Mac OS X systems, the Tizen SDK requires the Xcode command-line tools. See Installing the iOS SDK: Installing Command Line Tools for detailed instructions.

For Ubuntu 12.04, run the following command to install extra prerequisite packages:

sudo apt-get install procps gettext libdbus-1-3 libcurl3 expect gtk2-engines-pixbuf grep zip make qemu-user-static libwebkitgtk-1.0-0

Download the the Install Manager and SDK Image for your OS from https://developer.tizen.org/downloads/tizen-sdk. For Linux and Windows, only use the 32-bit versions.

If you are running Mac OS X 10.9 (Mavericks) or Windows 8.1, you need to install the hotfix from Intel after installing the SDK with HAXM: http://software.intel.com/en-us/articles/intel-hardware-accelerated-execution-manager/.

Installation

On Windows, your Tizen SDK directory cannot contain spaces in the path. Currently, the batch files do not support paths with spaces. This is an issue with the Tizen SDK. The default install location does not contain a path with spaces.

Install the SDK

  1. Once the SDK has been downloaded, run the Install Manager.

  2. When the Install Manager dialog appears, click the Advanced button.

  3. Select the SDK image radio and browse the the image you downloaded above. Click OK.

  4. Click the Next button.

  5. Select the components you want to install and click the Next button.

  6. Select the SDK home folder and click the Install button. Make a note of the path to the SDK home folder. This is a configuration setting for Studio. By default, this is ~/tizen-sdk on Mac OS X and Linux, and C:\tizen-sdk on Windows.

Install the Intel Hardware Accelerated Execution Manager (HAXM)

This step can be skipped for Linux systems, as hardware acceleration is usually enabled on Linux by default. Without hardware acceleration, the Tizen emulator runs very slowly even on the fastest machines. It is recommended to configure hardware acceleration. Important: HAXM for Tizen conflicts with HAXM for Android. You might have to uninstall the Android version first.

By default, HAXM is installed by the Tizen SDK. If it was not or you wish to manually install it, download the Hardware Accelerated Execution Manager from https://developer.tizen.org/downloads/sdk/installing-sdk/hardware-accelerated-execution-manager. Run the installer. For Windows systems, reboot your computer.

Create an Emulator

  1. Run the emulator-manager, which is located in the tizen-sdk/tools/emulator/bin directory.

  2. In the tree pane, under tizen2.x > x86-standard, select << Create new ... >>, then click the Create button.

  3. Ensure that the following options are set:

    • Display Resolution: 480x800 (might vary)

    • RAM Size: 512 or 1024 MB

    • HW Virtualization: Supported – If disabled, check if the Intel HAXM for Tizen program is installed properly. Review the directions from Intel Hardware Accelerated Execution Manager.

    • HW GL Acceleration: Supported

  4. Enter a name for your emulator in the Name textbox, then click the Confirm button.

  5. Click the Launch button to start it.

Create an Author Certificate

An author signature determines the integrity of an application package as intended by the developer and confirms that all applications that are signed with same author certificate are trustworthy. The author certificate, which is used for signing an author signature, must be generated with the shell or command-line tool in the SDK, before Tizen development can start.

The certificate generator tool is present in the tizen-sdk/tools/certificate-generator directory. To run the tool, execute the certificate-generator.sh or certificate-generator.bat file with the shell or command line. The tool interactively prompts for all the required information.

The tool outputs a PKCS #12 file, which is generated based on a set of input parameters. The generated private key and certificate are stored in the PKCS #12 file. Remember the path to your PKCS #12 file and password for Studio configuration.

Register an Author Certificate

The generated certificate must be registered with Tizen. To register the certificate:

  1. Launch the Tizen IDE.

    • On Linux, execute the tizen-sdk/ide/startup.sh script.

    • On Mac OS X, open the tizen-sdk/ide/TizenIDE.app file.

    • On Windows 7, click Start > All Programs > Tizen SDK > Tizen IDE.

    • On Windows 8, open the tizen-sdk/ide/IDE.exe file.

  2. Open the Preferences dialog. In the menu bar:

    1. For Linux and Windows, select Window > Preferences.

    2. For Mac OS X, select Tizen IDE > Preferences.

  3. Select Tizen SDK > Secure Profiles.

  4. Click the Add button and provide any name for the security profile. Click OK.

  5. Under the Profile Items section, in the Author Certificate section:

    1. In the Certificate path field, provide the path to the PKCS #12 file.

    2. In the Pass field, enter the password you used when generating the certificate.

  6. Click Apply then OK.

Configuring Studio

These steps outline the Studio Preferences settings that need to be changed to support Tizen development.

  1. Start Studio.

  2. From the menu, select Windows > Preferences on Window and Linux, or select Titanium Studio > Preferences or Appclerator Studio > Preferences on Mac OS X.

  3. In Preferences, select Studio > Platforms > Tizen.

  4. In the Tizen SDK Home field, enter the path to tizen-sdk.

  5. In the Keystore field, enter the path to your PKCS #12 file, when you generated an author certificate.

  6. In the Keystore Password field, enter the password for the author certificate.

  7. Click the OK button.

Studio is now setup to use the Tizen SDK.

Using Studio

On Windows, your workspace directory cannot contain spaces in the path. Currently, the batch files do not support paths with spaces. This is an issue with the Tizen SDK.

Tizen projects do not require any special templates or instructions compared to Android, iOS or Mobile Web projects. Use Studio the same as you would for any of these projects. The only exception is that the Tizen emulator needs to be started and running before building the application.

Updating the Tizen SDK

To update the Tizen SDK:

  1. Launch the Install Manager.

    • On Linux, run tizen-sdk/install-manager/InstallManager.

    • On Mac OS X, open the tizen-sdk/install-manager/InstallManager.app file.

    • On Windows 7, click Start > All Programs > Tizen SDK > Install Manager.

    • On Windows 8, open the tizen-sdk/install-manager/InstManager.exe file.

  2. Click Advanced.

  3. Select either Package server to update from the download server or SDK image to update using an image file, the click OK.

  4. Click Next and following the remaining directions.

Using the Tizen SDK

This section covers some of the important and useful tools distributed with the Tizen SDK. For more information about the Tizen SDK tools, see https://developer.tizen.org/documentation/dev-guide and navigate to Getting Started with Tizen > Development Environment > Tizen SDK.

Launching the Emulator

To launch a previously created emulator, use the emulator-manager tool, located in tizen-sdk/tools/emulator/bin directory, with the launch -n <emulator_name> option. For example, the following Linux command launches the emulator that was named default:

./emulator-manager launch -n default

Accessing Console Messages

To access console messages generated by Ti.API.info and similar calls, use the sdb tool located in the tizen-sdk/tools/ directory. For example, the following Linux command filters out all debug messages except the console messages:

sdb dlog | grep I\/ConsoleMessage

Troubleshooting

Windows: tizenapp.wgt was unexpected at this time

On Windows, the application failed to install to the emulator because the path to the project or SDK contains spaces. Currently, the batch files do not support paths with spaces. This is an issue with the Tizen SDK.

Reference: https://bugs.tizen.org/jira/browse/TSDK-75

Mac OS X: Cannot launch this VM. Shared memory is not enough.

If you installed the Tizen SDK for the first time, you may need to reboot the host machine and start emulator again. The maximum size of shared memory is changed after you reboot the host machine.

If the reboot did not work, run the following command in a terminal to adjust the shared memory:

sudo sysctl -w kern.sysv.shmmax=1610612736
sudo sysctl -w kern.sysv.shmall=393216
sudo sysctl -w kern.sysv.shmmin=1
sudo sysctl -w kern.sysv.shmmni=32
sudo sysctl -w kern.sysv.shmseg=8
sudo sysctl -w kern.maxprocperuid=512
sudo sysctl -w kern.maxproc=2048

You may need to run these commands after subsequent reboots.

Reference: https://bugs.tizen.org/jira/browse/TSDK-81

Mac OS X: The progress bar on Studio gets stuck at 59% when trying to install an application on device

The Tizen SDK sdb tool hung when trying to install your application on device. To workaround this issue:

  1. Disconnect your device from the host machine, that is, unplug the USB cable to the device.

  2. Run the following commands in a terminal to restart the sdb server:

    cd tizen-sdk/tools
    ./sdb kill-server
    ./sdb start-server

  3. Reconnect your device to the host machine.

  4. Run the following command in a terminal to confirm you device is connected. It should show your device ID:

    ./sdb devices

  5. In Studio, relaunch the application to build on device.
