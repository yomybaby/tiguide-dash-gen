Overview

This guide may help if there are problems with your Titanium Studio installation on Linux, after you have followed the Quick Start or Setting up Studio guides.

Common Problems

If your Eclipse installation is shared (installed via package manager into /usr/lib), give your login account root permissions on the /usr/lib/ecplise folder because running an update copies files to the /usr/lib/eclipse features and plugins directories. If you have Eclipse installed as a tarball into your home dir (or any writable area), you won't need to run as root.

Some users have found the following workaround helpful:

  1. Install 32-bit Sun Java (in Ubuntu: sudo aptitude install ia32-libs ia32-sun-java6-bin)

  2. Select as the Java version to be used (again in Ubuntu: sudo update-alternatives --config java and choose the 32-bit version)

Ubuntu 11.04/Natty Narwhal Menu Issues

Ubuntu 11.04 introduced the Unity window manager, which unfortunately wreaks havoc with the file menu. A bug discussion is here:

https://bugs.eclipse.org/bugs/show_bug.cgi?id=330563

In the meantime, create a Bash script that helps with a workaround:

#!/bin/bash
export UBUNTU_MENUPROXY=0
./AptanaStudio3

and execute this instead of Studio directly.

Ubuntu 12.04/Precise Pangolin Startup Issues

1. If you get an error similar to:

An internal error has occurred.
No more handlesNative code library failed to load. (java.lang.UnsatisfiedLinkError: /opt/Aptana_Studio_3/plugins/com.aptana.swt.webkitbrowser.linux.x86_64_1.0.0.1295409059/os/linux/x86_64/libcefjni.so: libjpeg.so.62: cannot open shared object file: No such file or directory)

Install necessary prerequisites:

sudo apt-get install libjpeg62 libwebkitgtk-1.0-0

2. If you get the following error:

java.lang.NullPointerException
at com.aptana.portal.ui.browser.AbstractPortalBrowserEditor.createPartControl(AbstractPortalBrowserEditor.java:118)
at org.eclipse.ui.internal.EditorReference.createPartHelper(EditorReference.java:670)
at org.eclipse.ui.internal.EditorReference.createPart(EditorReference.java:465)

Please make sure MOZILLA_FIVE_HOME is set as described in http://www.eclipse.org/swt/faq.php#browserlinuxrcp and then restart Studio. If the error persists, try installing xulrunner on the machine and following the steps below:

  1. The xulrunner can be downloaded from http://ftp.mozilla.org/pub/mozilla.org/xulrunner/releases/1.9.0.4/

  2. Unzip and extract the xulrunner to /usr/lib/xulrunner

  3. Run ./configure from xulrunner folder to register globally (so that SWT/Eclipse can recognize it).

  4. Restart Titanium Studio

Fedora

You must use Sun's JRE rather than the GNU JRE in order for several Aptana features to work correctly.

Error: The bundle could not be resolved. Reason: Missing Constraint (Fedora Core 8, IcedTea JVM)

Change your JRE; Fedora 9, 10 and 11 ships with GNU Java. To fix, create a logical link to an earlier version of Java (preferably JRE 1.6 from Sun).

ls -l /usr/local/aptana/jre
/usr/local/aptana/jre -> /usr/java/jre1.6.0_03
